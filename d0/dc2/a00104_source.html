51 # 前言
52 -----------
53 SEIMS（Spatially Explicit Integrated Modeling System）模型是由刘军志等人开发的以栅格为基本模拟单元、能体现水流空间运动的全分布式水文模型。SEIMS在借鉴Wetspa、LISEM、CASC2D、DHSVM和流溪河模型等多个模型基础上开发的模块化水文模型，采用C++编写，实现了子流域-基本单元的双层并行，采用的并行策略为共享内存（OpenMP）和消息传递（MPI），并利用NoSql数据库MongoDB进行数据组织管理。模型可在Windows及Linux平台下运行。
54 
55 目前，模型还不完善，主要存在以下主要问题：
56 
57 1. 生态和养分循环还很初步，距实用还相差很远；
58 2. 水文和侵蚀模块虽相对成熟，但尚缺乏进一步的验证工作。
59 
60 因此需要大家群策群力完善模型，主要有以下几个目标：
61 
62 1. 每个模块都具有完善的理论和开发文档，添加符合doxygen规范的代码注释；
63 2. 编码规范（包括模块和变量命名，编程习惯等），空闲了可以多看看Google C++ Code Style（[EN](http://google-styleguide.googlecode.com/svn/trunk/cppguide.html)，[CN](http://www.cnblogs.com/ggjucheng/archive/2012/01/02/2310278.html)）；
64 3. 编写模块需要与当前主流的其他模型进行对比验证，多做测试；
65 4. 为用户提供易用的Web界面，用户可选择在服务器端运行模型，也可以选择把模型和数据下载到本机运行。

82 ## 从Github中克隆代码库
83 配置好本地的Git环境之后，我们需要将代码库从Github上克隆到本地计算机。
84 + 首先在Github.com上注册一个账号。注册好之后，需要设置SSH key，在`git bash`中输入：
85 ~~~
86 ssh-keygen -t rsa -C youremail@example.com
87 ~~~
88 
89 一路回车，如果顺利的话，在用户目录里能找到`.ssh`目录，里面有`id_rsa`和`id_rsa.pub`两个文件，这两个就是SSH Key的密钥对，前者是私钥，后者是公钥。
90 + 然后，登录Github，打开 Account Settings，SSH key页面，点Add SSH Key，Title任意填，在Key文本框里填上`id_rsa.pub`的内容，点击Add key就好啦。
91 + 打开SEIMS模型的首页https://github.com/seims/SEIMS， 点击`Fork`，来创建自己账户下的SEIMS克隆，克隆结束之后，复制克隆库的SSH地址，这样它会自动使用你自己的SSH密钥，而不用每次在`git pull`或者`push`时询问你的用户名和密码。下一步，我们将克隆一份代码库到本地计算机。
92 
93 ~~~
94 cd <destination folder>
95 git clone git@github.com:<yourname>/SEIMS.git
96 or
97 git clone https://github.com/<yourname>/SEIMS.git
98 ~~~
99 
100 两种克隆方式推荐使用https，不用每次push时都要输入密码，如果用第一种方式，可以采用如下方式免输密码：
101 
102 ~~~
103 open git bash
104 cd
105 touch .git-credentials //创建.git-credentials文件
106 vim .git-creditials //用vim编辑此文件，输入以下内容
107 touch .git-credentials
108 vim .git-credentials
109 https://{username}:{password}@github.com
110 //然后输入
111 git config --global credential.helper store
112 //而后查看.gitconfig文件，发现多了一项：

114  helper = store
115 //重启git bash就会发现，git push时不用再输入密码了
116 
117 ~~~
118 
119 + 切换到克隆目录，在这里可以添加上游远程仓库：

122 ~~~
123 + 这样，我们就将SEIMS代码克隆到了本地计算机一份。接下来，我们尝试对本地SEIMS文件做一下修改，然后提交到远程库（你自己的），最后通过`pull request` 提交给SEIMS代码拥有者。
124

127 ~~~

129 Git commit –m "modification test”

131 ~~~
132

134

233 ## mongoDB
234 运行模型需要两个数据库源：气象数据库和参数数据库。
235 1. 气象数据库：一个存放有降水和气象数据的`mongodb`数据库
236 2. 参数数据库
237 一个以模型名称命名的mongodb数据库，包括下面几个表：
238 + parameters：包含模型的各个输入参数的信息，可以通过调整Impact参数进行调参
239 + reaches：河道相关信息
240 + SiteList：模拟所需的降水和气象站点
241 + Spatial：模拟所需的栅格数据、插值权重数据等，以`GridFS`方式存储
242 数据库的准备由预处理程序完成。

245 ## 模型配置文件
246 在每个模型的文件夹下包含三个主要的文件：
247 
248 + `config.fig`：包括模型运行所需的模块信息。模型运行时会按照顺序依次加载该文件中指定的各个模块（每行的最后一个以‘|’分隔的字符串），并顺序执行。
249 
250 通用格式为：
251 ~~~
252 No. | SettingString | | ModuleName
253 i.e.
254 0 | TimeSeries_P | | TSD_RD
255 if this, the corresponding module name should be TSD_RD_P so that the module name is unique.
256 ~~~
257 + `file.in`：包括模型运行的:
258  + MODE（`Daily`指定模型做日尺度的模拟；`Storm`指定模型做次降水的模拟）；
259  + Interval（步长，如果为Daily模拟，步长默认为1天，不可修改；如果为Storm模拟，步长单位为秒，可以分别指定坡面过程和河道过程的模拟步长，如“60, 60”指定次降水模拟的坡面和河道过程步长都是60s）；
260  + 模拟开始时间（格式：YYYY-MM-DD HH:MM:SS）
261  + 模拟结束时间
262 + `file.out`：指定模拟的输出，格式示例如下：
263  + OUTPUTID|T_PET
264  + SITECOUNT|1
265  + SITEID|53399
266  + STARTTIME|2009-12-30 00:00:00
267  + ENDTIME|2010-12-31 00:00:00
268  + FILENAME|T_PET_1.asc
269 ~~~
270 可选标签有：
271 OUTPUTID: 输出变量名
272 INTERVAL: INTERVAL is used only for the PET_TS output
273 SITECOUNT: is used only for the time series output for sites
274 SITENAME:
275 SITEID:
276 REACHNAME:
277 COUNT:
278 TYPE: 整合类型->
279  enum AggregationType
280  {
281  AT_Unknown = 0, ///< unknown
282  AT_Sum = 1, ///< sum
283  AT_Average = 2, ///< ave
284  AT_Minimum = 3, ///< min
285  AT_Maximum = 4, ///< max
286  AT_SpecificCells = 5 ///< output values of specific cells
287  };
288 DATA_TYPE:
289 WEIGHT:
290 VERTICALINTERPOLATION:
291 SUBBASINCOUNT: is used only for the time series output for subbasins
292 SUBBASINID:
293 RESERVOIRCOUNT:
294 RESERVOIRID:
295 STARTTIME: 开始时间
296 ENDTIME: 结束时间
297 FILENAME: 指定输出文件名及扩展名，如变量为Raster类型，则输出（.tif）文件
298 ~~~
299 
300  + 如果输出变量为"QOUTLET"、"QTotal"、"SEDOUTLET"、"DissovePOutlet"、"AmmoniumOutlet"、"NitrateOutlet"，则需要有STARTTIME、ENDTIME、FILENAME

303 ## 运行命令
304 + OpenMP调用格式为：
305 ~~~
306 seims_omp <ModelPath> <threadsNum> [<layeringMethod> <IP> <port>]
307 
308 <ModelPath> is the path of the configuration of the Model.
309 <threadsNum> must be greater or equal than 1.
310 <layeringMethod> 0 and 1, which means UP_DOWN and DOWN_UP respectively.
311 <IP> is the address of mongodb database, and <port> is its port number.
312 ~~~
313 ModelPath的文件夹名称需与mongodb中数据库名称对应，且文件夹内需包含`config.fig`、`file.in`、`file.out`。其中`config.fig`指定模型的算法选择，`file.in`指定模型模拟尺度和模拟时段，`file.out`指定模型输出。
314 + MPI调用格式为：
315 ~~~
316 mpiexec –n 6 seims <parallel.config file>
317 -n 指定并行计算的核数
318 <parallel.config file> 为配置文件，包含
319  ModelName，ProjectPath，ModulePath，DBHost，DBPort，TreadNumber，LayeringMethod
320 ~~~
321 与OpenMP版本一样，ProjectPath内也需有`config.fig`、`file.in`、`file.out`。

324 # SEIMS基本框架
325 在SEIMS模型中，每个子过程模拟方法对应一个独立的模块，并以动态链接库的形式存在（Windows 系统中的`DLL`文件，Linux系统中的`so`文件）。多个子过程模块构成子过程模块库，在应用时可以从模块库中选择一系列相互配合的子过程模块以动态组合运行的方式完成水文模拟任务。为了保证各个模块具有相同的接口，SEIMS模型中定义了一个模拟功能的父类[`SimulationModule`](../src/base/util/SimulationModule.h)，所有模块中都必须包含一个继承该父类的子类，并实现相应的接口函数（见下表）。
326 
327 接口函数 | 功能
328 -------- | -------
329 MetadataInformation | 获取模块输入输出的元数据
330 SetDate | 设置时间
331 SetValue | 设置数值类型的输入
332 Set1DData | 设置一维数组类型的输入
333 Set2DData | 设置二维数组类型的输入
334 Set3DData | 设置三维数组类型的输入
335 GetValue | 获取数值类型的输出
336 Get1DData | 获取一维数组类型的输出
337 Get2DData | 获取二维数组类型的输出
338 Get3DData | 获取三维数组类型的输出
339 Execute |执行模块功能
340 
341 模块中包含的接口函数可分为三类：
342 
343 + 元数据获取函数
344 
345 `MetadataInformation`函数负责为主程序提供模块的输入输出等元数据信息。模块的元数据信息采用`XML（eXtensible Markup Language）`文档表达，包括如下几部分：
346  + 模块的基本信息：包括模块 ID、列表、作者、描述、版本等；
347  + 来源于水文数据库的静态输入信息：包括各个输入参数的 ID、类型、单 位、描述、来源数据库名称等；
348  + 来源于其他模型输出结果的动态输入信息：包括各个输入参数的 ID、类 型、单位、描述等；
349  + 模型的输出信息：包括各个输出参数的 ID、类型、单位、描述等。
350 
351 + 数据设置与获取函数
352  + 模块接口中的 `SetValue` 和 `SetXDData` 系列函数负责设置模块所需参数，而`GetValue` 和 `GetXDData` 系列函数负责获取模型的计算结果。模块中的变量类型包括时间、单个值、一维数组、二维数组和三维数组。为提高数据传递效率，数组类型的变量通过指针的方式进行数据传递，即不同模块中的同一个变量的指针指向内存中的同一块区域。
353 + 模拟功能的执行函数
354  + `Execution`函数执行一个时间步长的模拟功能，而其时间循环由主程序控制。

357 ## 模型运行流程
358 在子过程模块库的基础上，SEIMS模型的主程序（main模块）负责对一系列模块进行动态组合构建水文模拟工作流完成模拟任务，其总体运行流程如下：
359 
360 + 模型初始化
361  + 首先读取配置文件中参与模拟的模块列表、模拟时段及步长等信息；
362  + 根据配置信息动态加载所需模块并调用 `MetadataInformation`函数获取XML格式的元数据；
363  + 解析元数据信息构建模块之间的输入输出关系，对模块进行动态组合构建水文模拟工作流；
364  + 从数据库读取模拟所需数据，并调用各模块的相应接口函数为模块的输入变量赋值。
365 
366 + 水文模拟工作流的运行
367 
368 水文模拟工作流的运行包括一个时间循环，在每个时间步长按照初始化阶段构建的水文模拟工作流一次运行各模块即可。运行时需要在每个时间步长开始时更新降水、气温等驱动因子的输入。如果参与计算的模块时间步长不一致，需要在运行时调整各模块的运行次数。对于存在数据依赖关系且时间步长不一致的模块，需要对其输入输出进行调整。

373 # SEIMS模块编写流程
374 模块编写的一般步骤：
375 
376 + 阅读相关文献和模型文档，编写模型文档的理论部分；
377 + 设计模块名，基本命名规则为`模块功能_算法`，缩写规范自行查找，比如`YLD`可以是作物产量`Yield`的缩写，没有指定具体算法的，命名只有`模块功能`即可
378 + 根据模型的相关理论，参考已有成熟模型的代码，列出模型的输入（包括以其他模块的输出作为输入）、输出，编写元数据函数(`MetadataInformation`)代码
379 + 在头文件中定义所需变量，并做初始化、赋值和有效性检查的工作
380 + 初始化：构造函数末尾, `initalOutputs`(为本模块的输出变量开辟内存空间，并注意在析构函数中删除所开辟的空间)
381 + 赋值：`SetValue`, `Set1DData`, `Set2DData`
382 + 有效性检查：`CheckInputData`
383 + 根据公式，编写`Execute`函数的代码，实现模拟功能
384 + 编写输出代码，`Get1DData`，`Get2DData`等
