37 ### 1.3 Spatial Data
38 空间数据主要输入数据为DEM、流域出口、土壤粘粒砂粒含量及有机质含量、土地利用。
39 处理的步骤为：
40 + 调用[TauDEM](http://hydrology.usu.edu/taudem/taudem5/index.html)进行坡度、流向、河网提取、子流域划分等
41 + 根据SQLite数据库，通过土地利用和土壤属性及水文变量，进行模型参数提取，包括CN值、水力半径、曼宁系数、流速、IUH、潜在径流系数等等
42 + 最后，将所有栅格文件导入MongoDB数据库
43 ~~~
44 ## Spatial Data
45 SubbasinDelineation(np, WORKING_DIR,dem,outlet_file, threshold, mpiexeDir=MPIEXEC_DIR,exeDir=CPP_PROGRAM_DIR)
46 f = open(WORKING_DIR + os.sep + "ProjConfig.txt")
47 proj4Str = f.readlines()[2].strip()
48 f.close()
49 GenerateSubbasins(WORKING_DIR, proj4Str,exeDir=CPP_PROGRAM_DIR)
50 ## Extract parameters from landuse, soil properties etc.
51 ExtractParameters(landuseFile, sandList, clayList, orgList, WORKING_DIR, True, True, True, True)
52 ## Import to MongoDB database
53 BuildMongoDB(WORKING_DIR, SpatialDBName, stormMode, forCluster, ClimateDBName, PrecSitesVorShp,MeteorSitesVorShp)
54 ~~~

56 ## 2 Configuration for Preprocessing Program
57 以上输入数据准备完成之后，我们首先需要编译一下预处理需要的C++程序。
58 + 在源代码外新建一个build文件夹，如E:\build, 打开 “开始”->Microsoft Visual Studio 2010->Visual Studio Tools->Visual Studio 命令提示(2010)，以管理员方式运行，依次输入以下命令：
59 ~~~
60 cd e:\build
61 e:
62 cmake <SEIMS Folder>/preprocess
63 mkdir metis
64 cd .\metis
65 cmake <SEIMS Folder>/preprocess/cpp_src/metis-5.1.0-pk
66 ~~~
67 + 用VS打开`e:\build`文件夹下的`SEIMS_Preprocess.sln`，并生成解决方案，此时，预处理程序目录为`CPP_PROGRAM_DIR = r'E:\build\Debug'`，或者`CPP_PROGRAM_DIR = r'E:\build\Release'`
68 + 用VS打开`e:\build\metis`文件夹下的`METIS.sln`，并生成解决方案，此时，用于任务划分的`METIS`程序目录为`METIS_DIR = r'E:\build\metis\programs\Debug'`，或者`METIS_DIR = r'E:\build\metis\programs\Release'`
69 
70 编译完成后，打开`./preprocess/config.py`进行程序运行前的配置
71 + 首先是一些目录的指定，下面是`Windows`的设置，`Linux`也类似：
72 ~~~
73 ## Directionaries
74 if platform.system() == "Windows":
75  DATA_BASE_DIR = r'E:\data_prepare'
76  PREPROC_SCRIPT_DIR = r'<SEIMS Folder>\preprocess'
77  CPP_PROGRAM_DIR = r'E:\build\Debug'
78  METIS_DIR = r'E:\build\metis\programs\Debug'
79  MPIEXEC_DIR = None
80 ~~~
81 + 然后，设置`mongoDB`数据库相关,下例为本机数据库地址：
82 ~~~
83 ## MongoDB related
84 HOSTNAME = '127.0.0.1'
85 PORT = 27017
86 ClimateDBName = 'climate_dianbu'
87 SpatialDBName = 'model_dianbu_30m_longterm'
88 forCluster = True
89 stormMode = False
90 ~~~
91 `forCluster` 如为`True`，则说明所建数据库是为`MPI`并行程序准备的，此时，所有栅格数据均按照子流域编号进行分别存储，如坡度数据在`forCluster`为`False`时，数据库中存储为`1_SLOPE`,而`forCluster`为True时，则存储为`1_SLOPE`, `2_SLOPE`...`N_SLOPE`
92 
93 stormMode指示数据库是为暴雨模拟还是长时段模拟所建
94 
95 + 再然后，需要设置`Climate`数据的路径，其中`PrecDataYear`如果有多个年份，则可为[2011,2012,2013]等
96 ~~~
97 ## Climate Input
98 PrecSitesVorShp = CLIMATE_DATA_DIR + os.sep + 'shp' + os.sep + 'Preci_dianbu_Vor.shp'
99 if stormMode:
100  PrecStormSitesVorShp = CLIMATE_DATA_DIR + os.sep + 'shp' + os.sep + 'Preci_dianbu_Vor_storm.shp'
101 MeteorSitesVorShp = CLIMATE_DATA_DIR + os.sep + 'shp' + os.sep + 'Metero_hefei_Vor.shp'
102 PrecExcelPrefix = CLIMATE_DATA_DIR + os.sep + 'precipitation_by_day_'
103 PrecDataYear = [2014]
104 MeteoVarFile = CLIMATE_DATA_DIR + os.sep + 'Variables.txt'
105 MeteoDailyFile = CLIMATE_DATA_DIR + os.sep+ 'meteorology_dianbu_daily.txt'
106 MetroSiteFile = CLIMATE_DATA_DIR + os.sep + 'sites_hefei.txt'
107 DischargeExcelPrefix = CLIMATE_DATA_DIR + os.sep + 'discharge_by_day_'
108 DischargeYear = [2014]
109 ~~~
110 
111 + 随后，设置`SQLite`数据库名
112 ~~~
113 ## Parameters for SEIMS
114 sqliteFile = DATA_BASE_DIR + os.sep + "Parameter.db3"
115 ~~~
116 + 随后，设置空间数据路径及`TauDEM`运行的并行核数
117 ~~~
118 ## Spatial Input
119 dem = SPATIAL_DATA_DIR + os.sep + 'dem_30m.tif'
120 outlet_file = SPATIAL_DATA_DIR + os.sep + 'outlet_30m.shp'
121 
122 threshold = 0 # threshold for stream extraction from D8-flow accumulation weighted Peuker-Douglas stream sources
123  # if threshold is 0, then Drop Analysis is used to select the optimal value.
124 np = 4 # number of parallel processors
125 landuseFile = SPATIAL_DATA_DIR + os.sep + 'landuse_30m.tif'
126 sandList = []
127 clayList = []
128 orgList = []
129 for i in [1,2]:
130  sandFile = SPATIAL_DATA_DIR + os.sep + "sand" + str(i) + ".tif"
131  clayFile = SPATIAL_DATA_DIR + os.sep + "clay" + str(i) + ".tif"
132  orgFile = SPATIAL_DATA_DIR + os.sep + "org" + str(i) + ".tif"
133  sandList.append(sandFile)
134  clayList.append(clayFile)
135  orgList.append(orgFile)
136 defaultSand = 40
137 defaultClay = 30
138 defaultOrg = 2.5
139 ~~~
140 + 最后，可以选择根据土地利用和土壤属性查找表的项目，其中`LANDUSE_ATTR_LIST`为导入数据库时的表名，`LANDUSE_ATTR_DB`为`SQLite`数据库表中的属性列名
141 ~~~
142 # LANDUSE_ATTR_LIST and SOIL_ATTR_LIST is selected from sqliteFile database
143 LANDUSE_ATTR_LIST = ["Manning", "Interc_max", "Interc_min", "RootDepth", \
144  "USLE_C", "SOIL_T10","USLE_P"]
145 LANDUSE_ATTR_DB = ["manning","i_max","i_min", "root_depth", "usle_c", "SOIL_T10"]
146 SOIL_ATTR_LIST = ["Sand", "Clay", "FieldCap", "Conductivity", "Porosity",\
147  "USLE_K", "WiltingPoint", "Poreindex", "Residual", "Density"]
148 SOIL_ATTR_DB = ["sand", "clay", "fc", "ks", "porosity","usle_k","wp",\
149  "P_INDEX","rm","B_DENSITY"]
150 ~~~
