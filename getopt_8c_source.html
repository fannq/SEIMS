00125 /*

00128 If the caller did not specify anything, the default is REQUIRE_ORDER if 
00129 the environment variable POSIXLY_CORRECT is defined, PERMUTE otherwise.
00130 
00131 REQUIRE_ORDER means don't recognize them as options; stop option processing 
00132 when the first non-option is seen.  This is what Unix does.  This mode of 
00133 operation is selected by either setting the environment variable 
00134 POSIXLY_CORRECT, or using `+' as the first character of the list of 
00135 option characters.
00136 
00137 PERMUTE is the default.  We permute the contents of ARGV as we scan, so 
00138 that eventually all the non-options are at the end.  This allows options
00139 to be given in any order, even with programs that were not written to
00140 expect this.

00143 to expect options and other ARGV-elements in any order and that care

00214       top -= len;
