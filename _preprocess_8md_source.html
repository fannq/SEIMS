00034 ### 1.3 Spatial Data
00035 空间数据主要输入数据为DEM、流域出口、土壤粘粒砂粒含量及有机质含量、土地利用。
00036 处理的步骤为：
00037 + 调用[TauDEM](http://hydrology.usu.edu/taudem/taudem5/index.html)进行坡度、流向、河网提取、子流域划分等
00038 + 根据SQLite数据库，通过土地利用和土壤属性及水文变量，进行模型参数提取，包括CN值、水力半径、曼宁系数、流速、IUH、潜在径流系数等等
00039 + 最后，将所有栅格文件导入MongoDB数据库
00040 ~~~
00041 ## Spatial Data
00042 SubbasinDelineation(np, WORKING_DIR,dem,outlet_file, threshold, mpiexeDir=MPIEXEC_DIR,exeDir=CPP_PROGRAM_DIR)
00043 f = open(WORKING_DIR + os.sep + "ProjConfig.txt")
00044 proj4Str = f.readlines()[2].strip()
00045 f.close()
00046 GenerateSubbasins(WORKING_DIR, proj4Str,exeDir=CPP_PROGRAM_DIR)
00047 ## Extract parameters from landuse, soil properties etc.
00048 ExtractParameters(landuseFile, sandList, clayList, orgList, WORKING_DIR, True, True, True, True)
00049 ## Import to MongoDB database
00050 BuildMongoDB(WORKING_DIR, SpatialDBName, stormMode, forCluster, ClimateDBName, PrecSitesVorShp,MeteorSitesVorShp)
00051 ~~~

00053 ## 2 Configuration for Preprocessing Program
00054 以上输入数据准备完成之后，我们首先需要编译一下预处理需要的C++程序。
00055 + 在源代码外新建一个build文件夹，如E:\build, 打开 “开始”->Microsoft Visual Studio 2010->Visual Studio Tools->Visual Studio 命令提示(2010)，以管理员方式运行，依次输入以下命令：
00056 ~~~
00057 cd e:\build
00058 e:
00059 cmake <SEIMS Folder>/preprocess
00060 mkdir metis
00061 cd .\metis
00062 cmake <SEIMS Folder>/preprocess/cpp_src/metis-5.1.0-pk
00063 ~~~
00064 + 用VS打开`e:\build`文件夹下的`SEIMS_Preprocess.sln`，并生成解决方案，此时，预处理程序目录为`CPP_PROGRAM_DIR = r'E:\build\Debug'`，或者`CPP_PROGRAM_DIR = r'E:\build\Release'`
00065 + 用VS打开`e:\build\metis`文件夹下的`METIS.sln`，并生成解决方案，此时，用于任务划分的`METIS`程序目录为`METIS_DIR = r'E:\build\metis\programs\Debug'`，或者`METIS_DIR = r'E:\build\metis\programs\Release'`
00066 
00067 编译完成后，打开`./preprocess/config.py`进行程序运行前的配置
00068 + 首先是一些目录的指定，下面是`Windows`的设置，`Linux`也类似：
00069 ~~~
00070 ## Directionaries 
00071 if platform.system() == "Windows":
00072     DATA_BASE_DIR = r'E:\data_prepare'
00073     PREPROC_SCRIPT_DIR = r'<SEIMS Folder>\preprocess'
00074     CPP_PROGRAM_DIR = r'E:\build\Debug'
00075     METIS_DIR = r'E:\build\metis\programs\Debug'
00076     MPIEXEC_DIR = None
00077 ~~~
00078 + 然后，设置`mongoDB`数据库相关,下例为本机数据库地址：
00079 ~~~
00080 ## MongoDB related
00081 HOSTNAME = '127.0.0.1'
00082 PORT = 27017
00083 ClimateDBName = 'climate_dianbu'
00084 SpatialDBName = 'model_dianbu_30m_longterm'
00085 forCluster = True 
00086 stormMode = False
00087 ~~~
00088 `forCluster` 如为`True`，则说明所建数据库是为`MPI`并行程序准备的，此时，所有栅格数据均按照子流域编号进行分别存储，如坡度数据在`forCluster`为`False`时，数据库中存储为`1_SLOPE`,而`forCluster`为True时，则存储为`1_SLOPE`, `2_SLOPE`...`N_SLOPE`
00089 
00090 stormMode指示数据库是为暴雨模拟还是长时段模拟所建
00091 
00092 + 再然后，需要设置`Climate`数据的路径，其中`PrecDataYear`如果有多个年份，则可为[2011,2012,2013]等
00093 ~~~
00094 ## Climate Input
00095 PrecSitesVorShp = CLIMATE_DATA_DIR + os.sep + 'shp' + os.sep + 'Preci_dianbu_Vor.shp'
00096 if stormMode:
00097     PrecStormSitesVorShp = CLIMATE_DATA_DIR + os.sep + 'shp' + os.sep + 'Preci_dianbu_Vor_storm.shp'
00098 MeteorSitesVorShp = CLIMATE_DATA_DIR + os.sep + 'shp' + os.sep + 'Metero_hefei_Vor.shp'
00099 PrecExcelPrefix = CLIMATE_DATA_DIR + os.sep + 'precipitation_by_day_'
00100 PrecDataYear = [2014]
00101 MeteoVarFile = CLIMATE_DATA_DIR + os.sep + 'Variables.txt'
00102 MeteoDailyFile = CLIMATE_DATA_DIR + os.sep+ 'meteorology_dianbu_daily.txt'
00103 MetroSiteFile = CLIMATE_DATA_DIR + os.sep + 'sites_hefei.txt'
00104 DischargeExcelPrefix = CLIMATE_DATA_DIR + os.sep + 'discharge_by_day_'
00105 DischargeYear = [2014]
00106 ~~~
